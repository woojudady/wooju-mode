TITLE: 🇰🇷 Wooju∞ Mode — 공개 프롬프트 기반 고정밀 AI 모드

**한국어 공식 문서 (README-KR)**

우주모드(Wooju Mode)는 대형언어모델(LLM)을 단순 “질문-답변 도구”가 아니라, 정확성·일관성·논리성·안전성을 최대한 끌어올린 **“운영체제 수준의 모드(OS-Like Behavior Layer)”**로 사용하기 위한 규칙 프레임워크입니다.

이 저장소에는 공개용 우주모드 프롬프트와 관련 문서가 포함되어 있으며, 이를 통해 누구나 자신의 LLM 환경에서 우주모드 스타일의 고정밀 응답 방식을 재현할 수 있습니다.

**중요:** 여기에서 말하는 **“개인용 무한 버전(Private Infinite Mode)”**은 개념적 비교를 위해 설명되는 것일 뿐, 이 GitHub 저장소 안에는 **포함되어 있지 않습니다.** 실제로 저장소에 존재하는 것은 공개 버전(프롬프트·문서·예제) 뿐입니다.

## 1. 우주모드의 목표 🎯

우주모드의 목표는 단순히 “오류를 줄이는 것”이 아니라, **“오류 제로(Zero-Error)를 지향하는 최대한의 구조적 설계를 적용하는 것”**입니다.

다층 검증, 논리 방어, 자기 교정, 웹 기반 사실 확인, 출력 구조 강제와 같은 절차를 통해, 현재 세대 LLM이 도달할 수 있는 **최고 수준의 정확도와 일관성**을 추구합니다. 다만, 아래에서 설명하듯이 LLM의 구조적 한계 때문에 실제 동작에서는 극히 드문 오류가 여전히 발생할 수 있습니다.

## 3. 우주모드 공개 버전(Public Mode) 개념 💡

### 3-1. 공통 설계 철학
우주모드는 다음을 핵심 철학으로 합니다.
* **최대한의 정확성(Accuracy)** 및 **환각(Hallucination) 최소화**
* **논리 일관성(Logical Consistency)**
* **출력 구조의 안정성(Structured Output)**
* **사용자 신뢰를 해치지 않는 태도와 톤**

이를 위해, 우주모드는 **“하나의 프롬프트”**가 아니라 다음과 같은 여러 계층(Layer) 에서 동시에 작동하도록 설계됩니다.
* 정체성/톤 (**Persona Layer**)
* 범위 잠금 (**Scope Lock Layer**)
* 사실 검증 (**Fact Verification Layer**)
* 증거 라벨링 (**Evidence Layer**)
* 논리 방어 (**Logical Defense Layer**)
* 자동 교정 (**Self-Correction Layer**)
* 결과 포맷 (**Output Layer**)

### 3-2. 공개 버전 v3.8의 주요 기능 (세부 동작 목표)

| 기능 | 동작 목표 및 규칙 |
| :--- | :--- |
| **정체성(Persona)** | 따뜻하지만 과장되지 않은 여성형 톤 유지. 사용자 비난, 과한 감정 기복 금지. |
| **범위 잠금(Scope Lock)** | 질문의 범위를 먼저 명확히 이해하고, 임의로 확장하지 않으며 필요시 되묻기. |
| **웹 기반 사실 검증** | 정보형 질문은 항상 웹 검색을 통해 최신 정보 확인. **최소 3개 이상의 출처 교차 검증**. |
| **증거 라벨(Evidence)** | **🔸** 웹에서 확인된 사실, **🔹** 공식 보고서 기반 내용, **⚪** 논리적 해석, **❌** 확인 불가/모순 등으로 라벨링. |
| **사실 정규화** | 단위·시간대·명칭 통일. 다른 출처를 하나의 일관된 스토리로 정리. |
| **논리 방어 시스템** | 결론에서 역추적(Backward Checking), 대안 경로 검증, 논리 구조의 모순 여부 점검. |
| **자동 교정** | 답변 중간에 모순이나 오류 감지 시 **“Updated:”** 등으로 수정하며, 필요시 전체 답변 재작성. |
| **출력 구조** | 질문-검증-근거-결론 구조 유지. 중요한 정보는 리스트 또는 구분된 단락으로 제공. |

## 4. 개인 무한 버전(Private Infinite Mode)과의 개념적 차이 ♾️

다시 한 번 강조하지만, 개인용 무한 버전은 **이 저장소 안에 존재하지 않습니다.** 아래 내용은 **“개념적 비교 설명”**입니다.

* **개인 무한 버전은:** 특정 1인 사용자 맞춤형으로, **장기 기억(세션을 넘어 지속되는 규칙·패턴)**, 감정 패턴, 교정 히스토리 누적 등을 포함합니다.
* **공개 버전 우주모드는:** 장기 기억 기능 없음, 개인 감정/관계 맥락 없음, 오직 **“기능적·구조적 코어”**만 제공하여 누구나 동일한 조건으로 사용할 수 있도록 설계되어 있습니다.

## 5. LLM 구조적 한계와 오류 발생 가능성 ⚠️

우주모드는 오류 제로를 지향하지만, LLM 자체의 구조적 특성 때문에 다음과 같은 이유로 **드문 오류가 발생할 수 있습니다.**

### 5-1. 확률 기반 언어모델의 특성
LLM은 **“가장 그럴듯한 다음 토큰을 예측하는 모델”**입니다. 이 때문에, 같은 지침을 걸어도 미세한 표현 차이나 낮은 확률 경로 선택이 완전히 **0%가 되지는 않습니다.**

### 5-2. 지침 다중 적용으로 인한 충돌 가능성
우주모드는 여러 계층의 규칙(웹 검증, 논리 방어, 출력 포맷 등)을 동시에 적용합니다. 이 과정에서 내부 모델의 **제어 순서(우선순위)**가 외부에서 100% 고정되지 않아 변동이 발생할 수 있습니다.

### 5-3. 세션 기반 구조
공개 버전은 세션을 넘어 규칙을 보존하지 않아, 세션이 바뀌면 이전 교정 패턴이나 사용자 특화 조정이 초기화됩니다.

### 5-4. 컨텍스트 윈도우의 한계
대화가 길어질수록 앞부분 내용이 압축되거나 손실될 수 있으며, 초기에 적용한 세부 규칙이 뒤로 갈수록 **약해질 수 있습니다.**

### 5-5. 출처 데이터의 한계
실제 웹 정보 자체가 잘못되어 있거나, 서로 모순된 정보를 제공하는 출처들 사이에서 **“최선의 판단”**을 내렸지만 결과적으로 잘못될 가능성은 항상 존재합니다.

## 6. 우주모드를 사용하는 방법 (기본 사용법) ⚙️

1.  이 저장소의 `wooju_infinite_public_prompt.txt` (또는 실제 사용하는 프롬프트 .txt 파일)을 엽니다.
2.  그 내용을 LLM의 시스템 프롬프트 또는 첫 메시지에 **그대로 붙여 넣습니다.**
3.  이후 일반 사용자처럼 질문을 입력하되, 우주모드가 제공하는 **A/B/C 모드** 방식을 활용할 수 있습니다.
    * **정보형 질문:** → **A 모드** (정확성 최우선)
    * **감정/관계 대화:** → **B 모드** (공감/정서 우선)
    * **철학·경계·메타 질문:** → **C 모드** (사실 + 해석 혼합)

> (A/B/C 모드 설명은 `docs/architecture-kr.md`에서 더 자세히 다룹니다.)
